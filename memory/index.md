# Memory index (read first)

Minimal compound memory for agents. Prefer this over re-scanning the whole repo.

## Where to look

| Need | Path |
| --- | --- |
| Product / feature loop status | `memory/product-loops.md` |
| Working / broken paths | `memory/status.md` |
| Pointers to long docs | `references/index.md` |
| Active plan + backlog | `tasks/to-do.md` |
| Failures / wins to reuse | `lessons/index.md` → linked notes |

## Agent protocol (always)

1. **Plan default** — before coding, update `tasks/to-do.md` (plan section). Do not skip planning.
2. **Before work** — skim this index + relevant memory/lesson links.
3. **During work** — if a product/feature loop changes, or something works / fails / goes amiss → update `memory/*` and add a lesson under `lessons/`.
4. **After work** — tick/move items in `tasks/to-do.md`; keep files short (bullets, no essays).
5. **Token rule** — append facts, not narrative. Link to `docs/` or `UPDATES.md` instead of copying.
